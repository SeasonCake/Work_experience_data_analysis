## 项目一：承包商人员准入自动化系统

### 项目信息
| 项目 | 说明 |
|------|------|
| 名称 | Contractor Access Automation System |
| 仓库 | https://github.com/SeasonCake/Work_experience_data_analysis |
| 目录 | `/Contractor-Access-Automation` |
| 代码量 | ~2,500行 Python |
| 开发周期 | 独立完成 |

### 业务背景
大型化工/制造业项目中，承包商人员管理面临的挑战：
- 人员规模大（3000+人），手工核查耗时长
- 黑名单比对依赖Excel VLOOKUP，易遗漏
- 证书到期监控依赖月底手工统计，响应滞后
- 培训合规记录分散，难以追溯

### 技术方案

```
┌─────────────────────────────────────────────────────────────┐
│                    系统架构                                  │
├─────────────────────────────────────────────────────────────┤
│  数据层    │  Faker模拟生成 → Excel存储                     │
│  业务层    │  AccessController → 五维度检查引擎             │
│  展示层    │  Plotly → HTML交互仪表板                       │
│  输出层    │  Excel报表 + HTML可视化                        │
└─────────────────────────────────────────────────────────────┘
```

### 核心功能模块

| 模块 | 文件 | 功能 | 技术亮点 |
|------|------|------|---------|
| 数据生成 | `data_generator.py` | 模拟3000+承包商数据 | Faker库、季度差异化培训数据 |
| 准入控制 | `access_control.py` | 五维度检查引擎 | O(1)黑名单查找、多级预警 |
| 可视化 | `dashboard.py` | 交互式仪表板 | Plotly Treemap、趋势图 |
| OCR模拟 | `ocr_simulator.py` | 证件识别模拟 | 正则表达式、格式校验 |

### 五维度准入检查

```
人员信息 → [1.黑名单] → [2.证书有效期] → [3.证书真伪] → [4.培训状态] → [5.工种资质] → 结果
              ↓             ↓              ↓             ↓              ↓
           Hash O(1)     日期计算      格式校验      关联查询       规则匹配
```

## 项目二：隐患排查与风险可视化系统

### 项目信息
| 项目 | 说明 |
|------|------|
| 名称 | Hazard Risk Dashboard |
| 仓库 | https://github.com/SeasonCake/Work_experience_data_analysis |
| 目录 | `/Hazard-Risk-Dashboard` |
| 代码量 | ~3,200行 Python |
| 开发周期 | 独立完成 |

### 业务背景
工业安全管理中隐患数据分析的痛点：
- 隐患数据分散在周报中，难以识别高风险区域
- 整改率统计依赖手工汇总，无法实时监控
- 缺乏多维度风险分析视角（区域×类型×时间）

### 技术方案

```
┌─────────────────────────────────────────────────────────────┐
│                    系统架构                                  │
├─────────────────────────────────────────────────────────────┤
│  数据层    │  加权随机生成 → CSV存储 + JSON配置             │
│  分析层    │  RiskAnalyzer → 统计/趋势/帕累托分析           │
│  可视化层  │  Visualizer → 热力图/3D曲面/Treemap           │
│  展示层    │  DashboardGenerator → HTML交互报表            │
└─────────────────────────────────────────────────────────────┘
```

### 核心功能模块

| 模块 | 文件 | 功能 | 技术亮点 |
|------|------|------|---------|
| 数据生成 | `data_generator.py` | 模拟4500+隐患记录 | 加权分布、季节因子、整改逻辑 |
| 风险分析 | `risk_analyzer.py` | 多维度统计分析 | 帕累托分析、趋势计算 |
| 可视化 | `visualizer.py` | 6种图表类型 | 多边形热力图、3D曲面 |
| 仪表板 | `dashboard_generator.py` | HTML报表生成 | 客户端排序、分页加载 |

### 可视化亮点

| 图表类型 | 用途 | 技术实现 |
|---------|------|---------|
| 厂区风险热力图 | 9区域多边形风险分布 | Plotly Scatter + 4级渐变色 |

## 面试常见问题准备

### Q1: 为什么选择Python做这个项目？
> Python生态丰富，Pandas处理表格数据高效，Plotly支持交互式可视化，Faker可以快速生成测试数据。对于数据分析类项目，Python是最佳选择。

### Q2: 如何保证黑名单查找的效率？
> 使用Python的`set`数据结构存储黑名单身份证号，查找时间复杂度为O(1)，即使黑名单有上万条记录，单次查找也是常数时间。

### Q3: 为什么用Faker生成数据而不是用真实数据？
> 1. 保护隐私：真实数据涉及个人信息，不适合公开
> 2. 可复现：固定种子确保每次生成相同数据
> 3. 可控性：可以模拟各种边界情况（证书过期、培训不合格等）

### Q4: 帕累托分析在实际工作中有什么用？
> 帕累托法则（80/20法则）帮助识别"关键少数"。例如发现80%的隐患来自20%的类型（如"行走安全"和"消防隐患"），可以优先分配资源解决这些高频问题。

### Q5: 这个项目如何扩展到生产环境？
> 1. 数据层：Excel改为数据库（SQLite/PostgreSQL）
> 2. 接口层：添加REST API（Flask/FastAPI）
> 3. 部署：容器化（Docker）+ 云部署
> 4. 监控：添加日志、告警机制
